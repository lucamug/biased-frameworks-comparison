Elm - Base

The base of Elm is very small because the entire app is written in Elm. This is just to start the Elm application.

The div is empty and used just as placeholder.

The only two attributes that can be passed to the init method of Elm are the "node" where to attach the application and the "flags" that is just data that is passed to Elm.

Vue - Base

This is the entire Vue application. I replaced the majority of code with "..." just to make it comparable with the Elm base.

The div here contain actually the entire view, while "data", "computed" and "created" containt the rest of the application.

"el" is the analogue of "node" in Elm.


        

        

            

        

    

    

        

            
Elm - View

            
This view in Elm is much more spread vertically and more verbose.

            
The loop to iterate all products is made with "List.map: (a -> b) -> List a -> List b" while in Vue is done by v-for="product in products".

            
Note that the attribute "type" require an "_" at the end because it is a reserved word in Elm (used to create types).

        

        

            
Vue - View

            
This view in Vue is much more dense.

            
"v-for" is used to iterate all products. In Elm we used "List.map" for the same purpose.

            
"v-model" create a two-way binding. it is like "v-bind:value" + "v-on:input". Elm use only one-way binding so it requires both "onInput <| ChangeQuantity product.id" and "value <| String.fromInt product.quantity"
